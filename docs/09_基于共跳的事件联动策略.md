# 策略名称

基于共跳的事件联动策略

## 策略描述

普通相关性说的是“平时一起动”，而共跳说的是“在同一个消息冲击下，一起突然跳”。

论文特别强调，共跳比普通相关性更适合描述新闻驱动市场。  
所以这个策略专门研究：

- 哪些市场经常在同一个时刻被同一类消息打动

然后做联动补涨补跌。

## 策略举例

例子 1：

- BTC 监管利好消息先打到 BTC 相关事件市场
- 几秒或几分钟后，ETH 相关市场才开始反应
- 如果历史上这两个市场经常共跳，就可以提前做 B 市场

例子 2：

- 某条链升级成功
- 主事件市场先跳
- 生态项目相关市场稍后跟涨
- 这时可以做后反应的那一边

## 策略获利方式

它赚的是“消息扩散速度不一致”。

也就是说：

- 有些市场先反应
- 有些市场后反应
- 只要这种时间差稳定存在，就能拿来做联动套利

## 适用市场

- 同一条新闻能影响多个市场的场景
- 同一生态、同一监管主题、同一币种族群的市场

## 基础开仓条件

- A 市场出现显著跳跃
- B 市场历史上与 A 有较强共跳记录
- B 还没有同步反应

## 基础平仓条件

- B 完成补涨补跌
- 联动未发生并超过时间阈值
- A 的跳跃被证伪

## 风险提示

- 共跳关系可能只在特定阶段成立
- B 市场没反应不代表错了，也可能是本来就不该动
- 速度要求高，低频数据上效果可能被削弱
